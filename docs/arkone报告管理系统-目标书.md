# ArkOne 报告管理系统（小而精标准化产品目标书）

版本: v0.2
日期: 2026-02-06
状态: Draft
负责人: Bao

## 1. 产品定位
- 标准化检验报告管理工具
- 小而精、轻量可落地，满足核心场景且体验好用

## 2. 目标用户
1. 医院检验科/信息科
2. 第三方实验室对接人员

## 3. 目标范围（小而精）
### 必备能力
1. 标准数据模型与字段映射
2. 模板管理与版本管理
3. 报告预览、打印、PDF 输出一致
4. 模板编辑（结构化拖拽为主，高级 HTML 为辅）
5. 报告中心列表管理（预览/打印/下载）
6. 报告审核/发送状态展示

### 不做或后置
1. 用户权限体系
2. 多机构复杂权限
3. Word 模板导入
4. 复杂审批流

## 4. 关键体验目标
1. 新用户 10 分钟内完成首份标准模板
2. 模板预览与打印输出一致
3. 核心流程不超过 3 步（接入 → 模板 → 输出）

## 5. 核心流程（最短路径）
1. 进入报告中心列表
2. 预览/打印/下载报告

模板流程:
1. 进入模板中心列表
2. 新建或编辑模板
3. 预览与保存

## 6. 标准化产品关键点
1. 数据质量校验
- 必填字段缺失即提示
- 最少 3-5 条硬性校验（患者姓名、报告号、结果列表等）

2. 模板规范化
- 预设模板样式（至少 2-3 套）
- 结构化组件为主，减少自由度带来的不一致

3. 输出一致性
- A4 页面规则明确
- 打印与 PDF 的一致性校验

4. 版本化管理
- 模板版本号与描述
- 模板对比结果

5. 状态闭环展示
- 报告审核状态
- 报告发送状态

## 7. 体验边界
- 默认场景“开箱即用”
- 保留高级模式但不作为主路径
- 所有核心操作可追溯

## 8. 验收口径（小而精）
1. 用样例数据可生成预览、打印、PDF
2. 字段映射成功后可复用
3. 模板编辑可快速生成合规报告
4. 预览与打印一致
5. 报告状态展示正确

## 9. 迭代路线
### 版本 1（当前目标）
- 数据映射
- 模板编辑（结构化 + HTML）
- 模板预设与对比
- 报告中心列表与详情
- 预览、打印、PDF
- 审核/发送状态展示

### 版本 2（后续）
- 更强的规则引擎
- 模板导入/导出
- 多模板协作
